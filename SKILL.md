# 艾利特机器人 SEO/GEO 文章写作 + 智谱配图

本 skill 把「艾利特 SEO/GEO 文章写作提示词」与「智谱 GLM-Image 自动配图」打包为一个完整的内容生产流程：一篇文章从主题到带图成品，一次产出。

**直接产出富文本 HTML（可直接发布官网，单产物）**：官网 CMS 富文本编辑器只吃 HTML，直接粘 Markdown 会显示成纯文本。因此写作提示词第十四节要求模型**直接以语义化富文本 HTML 输出正文**（`<h2>/<h3>/<p>/<figure>/<table>` 等，结构与线上文章一致），长尾词 + SEO 信息放顶部 HTML 注释里。最终只交付**一个 `.html` 文件**，兼职/OpenClaw 把正文 HTML 粘进 CMS 正文框、把注释里的 SEO 字段填进 CMS 对应字段即可发布。**不产出 Markdown。**

**图片永久链接**：配图走阿里云 OSS（`--host oss`），输出永久 URL，兼职零操作；OSS_DOMAIN 必须留空以用 bucket 默认域名（自带 HTTPS）。

## 能力边界

- **写作部分**：完全遵循 `references/写作提示词.md`（艾利特官方写作规范，1720 行），不改动、不重写。覆盖 SEO/GEO、产品匹配、品牌口径、数据合规、内链白名单等全部约束。
- **配图部分（新增）**：调用智谱 GLM-Image，按 **B2B 科技信息海报风格**（3D 写实场景 + 信息卡片中文文字排版）生成**封面 1 张（4:3）+ 正文至少 3 张（1:1）**的带文字图，单篇最少 4 张。每张正文图与所在 H2 段落**强相关**（一段一图对应，禁用通用场景图）。封面含大标题/副标题/卖点图标，正文含小标题/关键标注——充分利用 GLM-Image「读懂指令、写对文字」的核心能力。

## 何时触发

- 用户给艾利特文章主题/关键词，要写 SEO/GEO 文章
- 用户要给艾利特文章配封面图、正文插图
- 用户说「写一篇关于协作机器人/码垛机器人/焊接工作站/人形机器人的艾利特文章」
- 用户要在艾利特官网博客/知识库发带图内容

## 前置条件：智谱 API Key

配图需要智谱开放平台 API Key。脚本按以下优先级自动读取环境变量：

1. `ZHIPU_API_KEY`
2. `BIGMODEL_API_KEY`

获取：登录智谱开放平台 → API Keys → 创建，然后 `export ZHIPU_API_KEY="你的key"`（或写进 `~/.bashrc`）。若缺失，配图步骤会跳过并提示配置；**文章写作不受影响**，会照常产出。

## 工作流（5 步）

### 第 1 步：写作（直接产出富文本 HTML）

读取 `references/写作提示词.md`，严格按其规范创作文章，包括：长尾关键词拓展（10-15 个）、SEO 信息（Title / Keywords / Description / 文章URL）、文章内容（引言 + H2/H3 正文）、FAQ（4-8 个）、总结、内链建议（仅用白名单绝对路径）。

**按写作提示词第十四节，正文直接以语义化富文本 HTML 输出**（`<h2>/<h3>/<p>/<figure>/<table>/<strong>/<a>` 等），不输出 Markdown。长尾关键词与 SEO 信息放在正文顶部的 HTML 注释 `<!-- -->` 里（发布时由编辑抄进 CMS 字段，不在页面显示）。

**写作部分一字不改地遵循原文提示词**，所有产品匹配、品牌口径、数据合规、禁用表达、内链白名单等约束全部生效。

### 第 2 步：配图规划

文章写完后，规划配图（**先列清单再逐张生成，严禁偷懒只配 1-2 张**）：

- **1 张封面图**：`1024x768`（4:3 横版），高度概括文章主题，强视觉冲击
- **至少 3 张正文配图**：`1024x1024`（1:1 正方形），每张对应一个具体 H2 段落；长文章有多个核心 H2 时可按「一段一图」增加（软建议单篇正文不超过 6-8 张，以免影响 Core Web Vitals）

配图位置：封面图放 SEO 信息块之后、引言段之前；正文配图嵌在核心 H2 段落（产品选型、工艺解析、行业应用、技术原理）正文之后，**一张图严格对应一个具体 H2**；最后一张放总结段之前。**配图必须与文章主题、所在段落强相关**——禁用「机器人在工厂作业」这类通用场景图，无合适场景的段落宁缺毋滥、不凑数。详见 `references/配图指南.md`。

### 第 3 步：构造配图 prompt

按 `references/配图指南.md` 为每张图构造**中文 prompt**（GLM-Image 中文理解强、文字渲染准，经实测标题/标注均无误）：

- 风格：**B2B 科技信息海报**（3D 写实场景 + 左侧信息卡片文字排版）
- 封面文字：大标题（文章标题精简版）+ 副标题（3-4 关键词用 · 分隔）+ 3 个卖点图标文字
- 正文文字：左上角小标题（对应 H2）+ 关键部位标注
- **强相关（硬规则）**：每张正文图只服务一个具体 H2 段落；prompt 的场景描述与 H2 标题**必须含该段落实关键词**（如「码垛」「焊接火花」「六维力传感器」），禁用通用场景图，不相关不输出
- **关键技巧**：所有要出现在图上的文字，prompt 里用**中文双引号 `""` 包裹**（GLM-Image 对引号内文字精准渲染，词准确率 0.91）
- 配色：科技蓝 + 工业灰 + 棕色 + 白

### 第 4 步：调用智谱 GLM-Image 生成

**脚本路径约定（重要）**：调用脚本必须用**绝对路径**——skill 目录即 `SKILL.md` 所在目录（OpenClaw 下通常是 `~/.openclaw/skills/elibot-seo-geo`）。智能体执行命令时的 cwd 是用户工作目录，不是 skill 目录，所以**不要写** `bash scripts/generate_image.sh` 这种相对路径，会找不到脚本。先确认 skill 目录的绝对路径再调用。

```bash
SKILL_DIR="$HOME/.openclaw/skills/elibot-seo-geo"   # 按实际安装路径改

# 生成封面（4:3，标准输出最后一行即图片 URL）
bash "$SKILL_DIR/scripts/generate_image.sh" --prompt "..." --size 1024x768

# 推荐加 --download 同步下载到文章同级 images/，规避智谱 CDN 约 24h 失效
bash "$SKILL_DIR/scripts/generate_image.sh" --prompt "..." --size 1024x1024 \
  --download "images/02-palletizing.png"
```

每张图调用一次，拿到智谱 CDN URL。封面 1 次（`1024x768`）+ 正文至少 3 次（`1024x1024`）= **单篇最少 4 次调用**。**强烈建议每次都带 `--download`** 把图片落到文章同级 `images/`，从源头规避链接失效对兼职的影响。

**图床中转（推荐，永久链接）**：若已配置阿里云 OSS（见 `references/智谱图像API.md` 第十节），加 `--host oss` 会自动把图上传到 OSS 输出**永久 URL**，兼职复制 MD 即可、图片永不过期：

```bash
bash "$SKILL_DIR/scripts/generate_image.sh" --prompt "..." --size 1024x768 --host oss --download "images/01-cover.png"
```

或在 `~/.bashrc` 设 `export ELIBOT_IMAGE_HOST=oss`，之后默认走 OSS。未配置 OSS 时默认 `none`（智谱直出）。OSS 上传失败会自动降级智谱 URL。

退出码：`0` 成功；`1` 参数/key/依赖错误；`2` 智谱 API 调用失败；`3` prompt 疑似内容审核拒绝；`4` OSS 上传失败（已降级输出智谱 URL）。

### 第 5 步：嵌入文章（富文本 HTML）

正文以富文本 HTML 输出，图片用 `<figure>` 嵌入：

```html
<figure><img alt="码垛机器人在物流仓库执行纸箱码垛作业" src="OSS永久URL"><figcaption>图1：码垛机器人在物流仓库的码垛应用示意</figcaption></figure>
```

- `alt` 文案包含核心关键词 + 图片内容描述（SEO 友好，落地写作提示词第五节"图片建议写明 alt"）
- 图注放 `<figcaption>`，编号连续，辅助阅读与 GEO 引用

文末附「配图说明」HTML 注释块（`<!-- -->`），标注图片链接类型（OSS 永久 / 智谱 CDN 24h）与兼职发布步骤，不显示在页面上。

### 第 6 步：富文本校验（可选）

正文已在第 1 步直接产出富文本 HTML，正常情况无需再转换。`scripts/md2html.py` 降级为**可选兜底/校验工具**，仅在以下情况使用：

- 模型偶发仍产出 Markdown（或拿到历史 MD 草稿），用它转成富文本 HTML 兜底；
- 需要从混合输入中只抽取正文、剔除 SEO 信息块/长尾词/内链建议/注释。

```bash
SKILL_DIR="$HOME/.openclaw/skills/elibot-seo-geo"   # 按实际安装路径改

# 兜底：把 MD 草稿转成富文本 HTML（仅当模型漏带 HTML 时）
python "$SKILL_DIR/scripts/md2html.py" article.md -o article.html
```

转换映射（结构与线上文章 `cms.jiasou.cn/news/*.html` 一致）：`##/###` → `<h2>/<h3>`、段落 → `<p>`、`![alt](url)+*图注*` → `<figure><img alt src><figcaption>`、`**加粗**` → `<strong>`、`[锚](url)` → `<a href>`、Markdown 表格 → `<table>`、列表 → `<ul>/<ol>`。纯标准库实现，兼容 python 3.x。

## 输出格式

最终产出**一个文件**：

**`文章名.html`（富文本正文，直接发布官网）** —— 写作提示词第十四节要求模型直接产出富文本 HTML，结构如下（不产出 Markdown）：

```html

长尾关键词拓展：

SEO Title：...
SEO Keywords：...
SEO Description：...

<h2>文章内容</h2>
<p>【引言段落】</p>
...正文（H2→<h2>，含至少 3 张正文配图 <figure>，每张对应一个具体 H2 段落）...

...
<h2>总结</h2>
<p>...</p>
```

- 内链建议：可作为正文末尾的一段 `<h2>内链建议</h2>` 富文本，或单独留给编辑处理（不混入 SEO 注释）。

> 写作部分严格遵循 `references/写作提示词.md` 第十四节「最终输出格式」（已改为直接产出富文本 HTML）。

## 重要约束

1. **写作部分严格不变**：所有写作决策走 `references/写作提示词.md`。配图是文章完成后的独立增强环节，不回改写作逻辑、不修改写作提示词原文。
2. **配图数量**：封面 1 张（`1024x768` 4:3）+ 正文**至少 3 张**（`1024x1024` 1:1），单篇最少 4 张；长文章可按「一段一图」增加，软建议单篇正文不超过 6-8 张以免影响 Core Web Vitals。**严禁偷懒只配 1-2 张**，必须先列配图清单再逐张生成。
3. **B2B 科技信息海报风格**：全篇视觉统一，3D 写实场景 + 信息卡片文字排版，科技蓝 + 工业灰 + 棕色 + 白。
4. **图片带正确中文文字**：充分利用 GLM-Image「写对文字」能力——封面带标题/副标题/卖点，正文带小标题/标注；prompt 里文字用中文引号 `""` 包裹。右下角「AI生成」为智谱平台强制水印（国内 AI 内容合规标注），发布时可保留或裁剪。
5. **配图强相关（硬规则）**：每张正文图严格对应一个具体 H2 段落；prompt 场景描述与 alt/图注必须含该段落实关键词（码垛/焊接火花/六维力传感器等），**禁用「机器人在工厂作业」这类通用场景图**；无合适场景的段落（纯参数对比、内链清单、FAQ、数据罗列）宁缺毋滥、不凑数。
6. **图片链接用 bucket 默认域名**：配图走阿里云 OSS（`--host oss`）输出永久链接。**`OSS_DOMAIN` 必须留空**，这样脚本用 bucket 默认域名（`ittx-ces.oss-cn-hongkong.aliyuncs.com`，自带 HTTPS）；若填自定义域名（如 `oss-cms.jiasou.cn`）且没配 SSL 证书，HTTPS 会裂图。详见智谱图像API.md 第十节。
7. **配图不替代产品实拍**：生成的插画是场景示意，涉及艾利特具体型号参数时，文字说明仍以官网为准（写作提示词的来源红线同样适用于配图说明）。
8. **产品匹配仍由写作提示词决定**：配图主体（协作/码垛/焊接/人形等）跟随写作提示词第四节「产品匹配逻辑」的结论，不另起炉灶。
9. **直接产出富文本 HTML**：官网 CMS 富文本编辑器只吃 HTML，不吃 Markdown。写作提示词第十四节要求模型**直接以富文本 HTML 输出正文**（不产出 Markdown）；长尾词 + SEO 信息放顶部 HTML 注释。最终只交付一个 `.html`，直接粘进 CMS 正文框发布。若偶发产出 Markdown，用第 6 步 `md2html.py` 兜底转换。
