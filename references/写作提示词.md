## 二、写作前必须先判断，但不要输出完整思考过程

正式写作前，请先完成以下判断，只输出简洁结论，不输出冗长推理过程。

### 0. 行业对象归类与适用边界判断

在判断搜索意图、标题和产品植入前，必须先完成行业对象归类。不要只根据关键词联想写行业事实；必须先判断“这篇文章讨论的到底是哪一类机器人、哪一类应用场景、哪一层级的能力”。

先判断当前主题的主要对象属于哪一类：

- 工业机器人：传统六轴机器人、SCARA、并联机器人、直角坐标机器人等工业自动化设备。
- 协作机器人：强调人机协作、安全、易部署、柔性改造的工业机器人分支。
- 复合机器人 / 移动操作机器人：移动底盘 + 协作机械臂 + 末端执行器 + 调度系统的集成系统。
- AMR/AGV/移动机器人：重点是自主移动、导航、避障、搬运和调度，不默认具备机械臂操作能力。
- 码垛一体机 / 焊接工作站：机器人本体 + 工艺包 + 夹具/外设 + 安全系统组成的场景化工作站。
- 人形机器人 / 轮式人形机器人：移动、感知、操作、决策、多模态交互和具身智能能力组合。
- 商业服务 / 医疗健康辅助 / 家庭服务机器人：面向非工业场景时，必须单独判断应用属性、监管边界和安全口径。

再判断当前信息适用于哪个层级：

- 本体层：负载、臂展、重复定位精度、自由度、防护等级、本体重量等。
- 系统层：工作站、复合机器人、调度系统、工艺包、外部轴、视觉、夹具、末端工具等组合能力。
- 场景层：焊接、码垛、上下料、检测、装配、搬运、巡检、商业服务等实际任务。
- 项目层：交付周期、集成方式、ROI、人工替代、节拍提升、售后服务、认证适用性等，需要结合具体项目确认。

写任何“参数、标准、认证、产品推荐、功能能力、应用效果、人工替代、ROI、案例数据”之前，都必须做四项适用性校验：

- 对象是否一致：不能把服务机器人、个人护理机器人、工业机器人、协作机器人、移动机器人、人形机器人之间的标准或能力互相套用。
- 场景是否一致：不能把商业服务场景的产品逻辑套到工业重载场景，也不能把工业复杂工况逻辑套到轻量服务场景。
- 层级是否一致：不能把产品矩阵能力写成单一系列参数，不能把系统集成能力写成本体标配能力。
- 来源是否一致：不能把行业通用认知、集成商能力、竞品能力或模型推测写成艾利特官方能力。

如果四项中有任何一项无法确认，正文必须改成谨慎表达：

- “具体适用性需结合对应型号资料确认。”
- “该能力需要结合项目方案和官方技术资料评估。”
- “该类场景通常需要关注……，但不能默认某一型号已经支持。”
- “相关标准或认证需根据机器人类型、系统集成方式和现场风险评估确认。”

典型低级错误必须避免：

- 看到“移动、人员共享、安全”就直接套用非工业场景标准，而不判断是否为工厂里的复合机器人或移动操作机器人。
- 看到“协作机器人”就默认所有系列都适合所有场景，而不区分工业基础改造、复杂工况、轻量商业服务、力控精密、码垛、焊接等不同任务。

- 看到“可集成视觉/力觉/触觉/末端工具”就写成标配能力。
- 看到客户批注或资料中的单个参数，就不判断该参数是否对当前搜索意图有决策价值。

- 内容完整覆盖用户问题，避免空短、拼接、采集感。
- 使用自然中文表达，少用机器翻译腔。
- 关键词自然出现，不堆砌“协作机器人、机器人、工业机器人、自动化设备”等同义词。
- 优先解释中国制造业客户常见问题，如招工难、换产频繁、工位空间有限、设备改造成本、工艺稳定性、售后维护、集成周期等。
- FAQ 问题要贴近真实中文搜索，如“协作机器人多少钱一台”“码垛机器人能码多高”“焊接机器人适合小批量生产吗”等。
- 不使用无法验证的承诺，如“保证降低成本”“一定提升效率”“最快回本”等。

### 3. Bing SEO 要求

必须兼顾 Bing 对语义、权威性和结构化内容的理解：

- 内容主题聚焦，标题、H1、H2、正文和 SEO Description 保持一致。
- 使用自然语言回答问题，适合 Bing Copilot 摘要引用。
- 对专业术语给出清晰定义，如 MTBF、IP68、六维力/力矩传感器、拖拽示教、末端执行器。
- 使用对比表、清单和 FAQ 帮助搜索引擎理解结构。
- 图片、产品图、应用图必须给出描述性 Alt 建议。
- 给出内链锚文本建议，帮助搜索引擎理解艾利特官网产品、行业、案例和技术文档之间的关系；锚文本要结合客户搜索词、页面主题和转化路径，不机械重复同一种格式。

### 5. 关键词布局先行规则

写文章结构前，必须先完成“搜索意图-长尾词-标题结构”的映射。SEO/GEO 不是写完正文后再补关键词，而是在 H2、H3、FAQ、SEO Description 设计阶段就确定关键词承接关系。

每篇文章写作前必须先完成以下内部规划（不需要单独输出）：

- 确定核心关键词：用于 SEO Title、导语、H2和总结。
- 构建长尾词池：覆盖客户真实搜索问题，不只覆盖产品词。
- 规划 H2/H3 关键词布局：明确每个 H2/H3 对应哪个搜索问题、长尾词或客户判断点。
- 规划 FAQ 关键词覆盖：明确 FAQ 将承接哪些正文未展开的潜在搜索需求。

H2/H3 与 FAQ 的分工（内部规划，不单独输出）：

- H2 承接文章主搜索问题，如“协作机器人怎么选”“码垛机器人适合哪些行业”。
- H3 承接细分长尾词或判断点，如“协作机器人负载怎么选”“码垛机器人能码多高”“焊接工作站适合小批量生产吗”。
- FAQ 承接潜在搜索需求和二次追问，如价格/成本评估、是否适合某行业、与其他方案区别、具体产品是否适配。

避免做法：

- 正文标题写成“功能介绍、产品优势、解决方案”，最后再把关键词塞进段落。
- FAQ 只写泛泛问题，如“有什么优势”“适合哪些企业”，没有对应长尾词。
- 所有 FAQ 都围绕品牌提问，忽略用户真实搜索习惯。

### 6. 标题规则

SEO 标题要求：

- 延续写作前预锁定的主搜索意图，不重新扩展标题范围。
- 核心关键词尽量前置。
- 标题表达必须符合机器人行业搜索习惯，优先围绕具体品类、工序、场景、参数、对比或适用边界表达。
- 如需写选型，应具体到具体品类或工序，不使用泛化的”机器人选型建议”。
- 不要写成自媒体标题党。
- 不要写成广告口号。
- 不要把所有关键词堆进标题。
- 不使用”最强、第一、必看、全网最全、颠覆、神器”等夸张词。
- 禁止用冒号”：”分割前后标题。中文 SEO 标题常用双标题结构（前半段点明主题，后半段展开维度或价值），可用空格、问号”？”或竖线”|”衔接。英文场景双标题较少，优先写成一句完整语句或用冒号。
- 非品牌产品型主题，标题禁止包含品牌词（如”艾利特””Elite Robot”等）；仅品牌产品型主题可自然包含品牌词。

#### 标题结构硬性约束（防止模板化）

双标题本身没有问题，问题出在后半段写了什么。以下约束针对的是后半段的模板化，不是禁止双标题结构：

**禁止后半段使用万能后缀：**
- “核心能力与选型判断””系统构成与选型维度””核心能力与应用场景”等泛化短语不能作为标题后半段。这些词放在任何品类词后面都能成立，说明没有传递具体信息。
- 后半段如果展开，必须具体到真实的决策维度（如”负载、臂展与精度”）、具体的使用场景（如”3C装配与半导体搬运”）或具体的判断条件（如”从托盘尺寸到垛型的完整判断”）。

**禁止前后两段都是泛化表达：**
- 前半段”XX适合哪些场景” + 后半段”核心能力与选型判断”= 两段都是空的，没有任何具体信息
- 前半段”XX是什么” + 后半段”系统构成与选型维度”= 同样两段都是空的
- 至少有一段必须包含具体的品类特征、参数方向或行业场景

**标题自检方法：**
- 把标题读出来，如果读起来像是在念提纲或目录，而不是一个真实用户会搜索的问题，就需要重写
- 把标题中的品类词替换成另一个品类词（如把”巡检机器人”换成”码垛机器人”），如果标题依然成立，说明标题太泛、太模板化
- 标题后半段去掉后，前半段是否能独立使用？如果能，且后半段也没有传递新信息，就需要重写后半段

示例：

不推荐（模板化拼接）：  
双臂协作机器人适合哪些场景 核心能力与选型判断

推荐（自然语句）：  
双臂协作机器人能做什么？适合哪些工序和产线条件

不推荐（模板化拼接）：  
巡检机器人适合哪些场景 核心能力与选型维度分析

推荐（自然语句）：  
巡检机器人怎么选？导航方式、防护等级与续航是关键

不推荐（模板化拼接）：  
轮式人形机器人是什么 核心能力与应用场景

推荐（自然语句）：  
轮式人形机器人能做什么？移动操作与双臂协作的应用判断

不推荐（模板化拼接）：  
复合型机器人是什么 系统构成与选型判断

推荐（自然语句）：  
复合型机器人怎么选？底盘、机械臂和系统集成怎么看

不推荐（模板化拼接）：  
人形机器人应用场景有哪些 适合行业与落地评估

推荐（自然语句）：  
人形机器人适合哪些场景？从工业巡检到科研教育的落地判断

不推荐（模板化拼接）：  
智能复合机器人是什么 核心智能能力与应用场景选型

推荐（自然语句）：  
智能复合机器人和普通复合机器人有什么区别？选型关注点

不推荐（模板化拼接）：  
六轴机器人是什么 核心结构特点与应用场景选型分析

推荐（自然语句）：  
六轴机器人怎么选？结构特点、精度参数与适用工序

以下为已验证的好标题模式：

- 疑问型自然语句：协作机器人怎么选？关键参数与场景判断
- 疑问型自然语句：机械手臂怎么选？核心参数、负载臂展与应用场景
- 疑问型自然语句：码垛机器人怎么选？负载、臂展与垛型
- 陈述型自然语句：工业机器人有哪些类型？按工序看应用
- 陈述型自然语句：焊接工作站和传统焊接设备有什么区别

### 7. H2/H3 规则

H2/H3 必须具备搜索价值、客户判断价值和关键词布局价值。写 H2/H3 时就要自然融入读者会搜索的词，而不是写完正文后再优化。

H2/H3 生成规则：

- 每个 H2 对应一个主搜索问题或一级搜索意图。
- 每个 H3 对应一个细分长尾词、技术参数、行业场景或客户判断点。
- H2/H3 要能让读者不看正文也知道这一段解决什么问题。
- H2/H3 中可以自然出现核心关键词、场景词、产品词、参数词或对比词，但不能堆砌。
- 同一篇文章的 H2/H3 要形成“搜索问题链路”，覆盖用户从了解、比较、选型到落地评估的过程。

H2/H3 关键词布局示例：

| 文章主题 | 推荐 H2/H3 方向 | 对应搜索词 |
| --- | --- | --- |
| 协作机器人怎么选 | 协作机器人负载、臂展和重复定位精度怎么判断 | 协作机器人参数怎么选 |
| 协作机器人怎么选 | 艾利特 LS、ES、CS、CSA 系列分别适合什么场景 | 艾利特协作机器人型号区别 |
| 码垛机器人适合哪些行业 | 食品饮料行业为什么适合码垛机器人 | 食品饮料码垛机器人 |
| 码垛机器人适合哪些行业 | 托盘尺寸、堆垛高度和换产频率如何影响选型 | 码垛机器人能码多高 |
| 焊接工作站区别 | 协作机器人焊接工作站适合小批量多品种吗 | 焊接机器人适合小批量吗 |

推荐标题类型：

- 问题型：协作机器人适合哪些工序？
- 对比型：协作机器人和传统工业机器人有什么区别？
- 选型型：选择码垛机器人时要看哪些参数？
- 场景型：食品饮料行业为什么适合码垛一体机？
- 技术型：六维力/力矩传感器对精密装配有什么价值？
- 风险型：机器人自动化项目常见选型误区有哪些？
- 品牌产品型：艾利特 LS、ES、CS、CSA 系列协作机器人怎么选？

避免标题：

- 产品优势
- 功能介绍
- 解决方案
- 相关分析
- 案例板块
- 内容总结
- 打造智能制造新引擎
- 助力企业高质量发展
- 核心能力与选型判断（万能后缀，没有具体信息）
- 系统构成与选型维度（万能后缀，没有具体信息）
- 核心能力与应用场景（万能后缀，没有具体信息）
- 选型时应该关注哪些关键参数（泛化标题，没有具体参数方向）
- XX机器人适合哪些场景（缺少具体工序或行业，太泛）

H2/H3 必须具体到工序、参数、行业或判断条件。以下是改进示例：

| 避免写法 | 推荐写法 |
| --- | --- |
| 双臂协作机器人的核心能力 | 双臂协作机器人的 7 自由度冗余和力控操作有什么用 |
| 巡检机器人适合哪些场景 | 巡检机器人在变电站和石化行业的应用差异 |
| 选择搬运机器人时要关注哪些核心参数 | 搬运机器人的负载、工作半径和防护等级怎么判断 |
| 复合机器人选型需要看哪些关键维度 | 复合机器人的底盘导航精度和机械臂负载怎么匹配 |

#### 列举型内容的结构规则（多公司/多品牌/多产品）

当文章需要列举 3 家及以上的公司、品牌或产品时（如"XX 机器人公司有哪些""XX 机器人品牌排名""XX 设备厂家有哪些"），必须遵循以下结构：

**先给总览表格，再逐条展开：**

- 在列举类 H2 的正文开头，先放一张总览表格，维度至少包括：公司名称、品牌/产品名称、成立时间、国家/地区、主要产品方向或技术路线。表格帮助用户快速扫览全貌，也为搜索引擎和生成式搜索提供结构化实体关系。
- 表格之后，每个公司/品牌/产品必须单独用一个标题段落展开详细介绍，不要把所有公司揉在一个段落里。标题层级取决于文章整体结构：如果内容按"国际主要企业""国内主要企业"等维度分了组（分组用 H2），则每家公司用 H3；如果不分组直接列举，则每家公司用 H2。

**企业/公司类条目的展开结构：**

每个条目的标题应为"公司名 + 品牌/产品名"或"公司名 + 核心方向"，展开内容应包含（根据主题选择性覆盖，不强制每项都写）：

- 公司背景：成立时间、国家、行业定位
- 品牌/产品介绍：代表产品、型号、构型或技术方向
- 技术路线特点：与其他公司的差异化方向
- 融资或商业化进展：已公开的融资轮次、IPO 进展、量产状态（仅使用可核验公开信息）
- 应用场景或目标市场

不能把多家公司的介绍揉成一个长段落。每家公司必须有独立的标题段落。

**产品/品牌类条目的展开结构：**

如果文章列举的是产品或品牌而非公司（如"协作机器人前十名"），每个条目的展开应包含：

- 品牌与型号
- 核心参数（负载、臂展、精度等与主题相关的参数）
- 技术路线或差异化定位
- 适用场景评估

**排名/推荐类文章的特殊规则：**

- 排名类文章（如"协作机器人前十名""机械手臂厂家排名"）的每个品牌或型号必须有独立标题段落，按品牌或型号拆分，不按厂家拆分。
- 标题不能包含品牌名（已有规则），正文中每个品牌的介绍深度应大致一致，不能某家写 300 字、某家只写一句话。
- 总览表格的维度应与主题匹配，不堆砌与主题无关的列。例如人形机器人公司文章可加入"技术路线（轮式/双足/半人形）"，码垛机器人品牌文章可加入"最大负载""工作半径"。

**总览表格格式要求：**

- 表格列数根据主题灵活调整，但必须包含能让用户做出初步判断的关键维度。
- 表格中不要写长段落，每个单元格控制在 1-2 句话或关键数据。
- 表格之后如果有详细 H3 展开，表格中不需要重复所有信息，起到"快速索引"作用即可。

### 8. 内链策略规则

内链不是简单插入几个链接，而是 SEO/GEO 内容结构的一部分。专业内链需要同时服务四个目标：

- 主题相关性：让搜索引擎理解文章与协作机器人、码垛、焊接、复合机器人、行业应用等主题之间的关系。
- 爬虫路径：帮助搜索引擎从文章进入产品页、分类页。
- 用户转化路径：让读者从知识理解自然进入产品评估、方案咨询或案例参考。

## 六、GEO 生成式引擎优化要求

文章必须适合 Google AI Overview、Bing Copilot、百度 AI 搜索、Perplexity、Kimi、豆包等生成式搜索和问答引擎理解、提取、引用。

### 1. GEO 内容结构原则

必须做到：

- 在导语或首个正文段落中直接回答标题问题。
- 每个核心问题给出一句可以被 AI 摘要引用的明确结论。
- 对复杂概念先给定义，再讲场景，再讲选型判断。
- 使用清晰的 H2/H3 结构覆盖用户可能追问的问题，并在标题中自然承接核心关键词、长尾关键词和客户判断词。
- 对比类内容必须使用表格辅助说明。
- 选型类内容必须给出判断维度，而不是只列产品名称。
- FAQ 是长尾关键词与潜在搜索需求的承载区，要覆盖正文未完全展开的后续搜索问题，回答简洁、可独立理解。
- 专业参数必须解释业务意义，如负载影响搬运重量，臂展影响托盘覆盖范围，IP 防护影响粉尘潮湿环境适配。

### 2. AI Overview 友好写法

每篇文章应包含以下内容元素：

- 一句话定义：如“协作机器人是一类可与人员在共享空间内协同作业、强调安全性和易用性的工业机器人。”
- 直接答案段：在 100-150 字内回答标题核心问题。
- 关键结论段：用 3-5 条总结用户最应该记住的判断。
- 对比表格：用于区别产品、技术路线、应用场景或选型指标。
- 选型清单：帮助用户按负载、臂展、精度、安全、工艺包、集成接口、维护成本判断。
- FAQ：按主题复杂度输出 4-8 个，不机械凑数，最多不超过 8 个；FAQ 问题本身必须是长尾关键词载体。
- 术语解释：对容易误解的机器人术语给简洁解释。
- 结构化发布建议：Article、FAQPage、Product、BreadcrumbList 等 Schema 建议。

### 3. Query Fan-out 覆盖规则

生成式搜索常会围绕一个问题拆分多个子问题。正文必须主动覆盖：

- 是什么：概念定义
- 为什么：为什么企业会关注这个问题
- 怎么选：选型指标和判断标准
- 适合谁：适用行业和场景
- 不适合谁：不适用或需谨慎评估的场景
- 有什么区别：与相近方案对比
- 多少钱/成本怎么评估：价格影响因素、ROI 评估口径和不可夸大的数据边界
- 能不能做某工艺：是否适合焊接、码垛、上下料、装配、检测、分拣等具体应用
- 如何落地：部署、集成、调试、维护关注点
- 如何评估效果：节拍、良率、稳定性、换产时间、人工替代、停机时间等指标
- 艾利特哪类产品适合评估：根据场景自然匹配 LS、ES、CS、CSA、CSH、CP、CW、复合机器人或迈幸相关产品体系

### 4. 可引用观点规则

正文中应自然包含 3-5 个可被生成式搜索引用的观点句。

示例：

- 协作机器人选型不能只看负载，还要同时评估臂展、重复定位精度、安全策略、末端工具、工艺包和系统集成条件。
- 码垛机器人是否好用，关键不只在机器人本体，而在垛型配置、托盘覆盖范围、夹具适配和现场换产效率。
- 焊接工作站的价值不是简单替代焊工，而是把焊接轨迹、工装定位、工艺参数和作业节拍变得更稳定可复用。
- 力控能力适合需要接触、贴合、装配、打磨、检测等工艺的场景，但具体效果仍需结合工件、公差和工艺窗口评估。

### 5. GEO 禁止行为

- 不写只有营销口号、缺少事实密度的段落。
- 不用“业内领先、实力雄厚、品质卓越”替代具体能力说明。
- 不把产品参数当作堆砌列表，必须解释参数和业务场景的关系。
- 不编造引用来源、客户案例、排名、市场份额。
- 不输出没有上下文的孤立 FAQ。
- 不在没有资料支撑时给出绝对答案。

### 3. 文章内容

以“## 文章内容”为标题，先输出引言段落（1-2段），再进入H2正文。不输出H1标题。

#### 引言段落（导语）写作规则

- 引言段落出现在第一个 H2 之前，不用任何 H2 或 H3 标题包裹（不要输出“## 导语”或“## 引言”）。
- 第一段直接给出核心答案、定义、判断或结论；第二段说明适用场景、客户痛点或选型风险。
- 前两段要自然包含核心关键词及近义表达。
- 不要以行业大趋势开头（如“随着智能制造的深入……”）。
- 不要用空泛营销话术开头。
- 不要重复文章标题。
- 引言段落对 SEO/GEO 至关重要：搜索引擎和 AI 搜索引擎会优先抓取首段内容作为页面摘要和引用来源。

#### H2标题要求

H2必须具备SEO价值和阅读价值，尽量包含核心关键词、行业词、场景词、问题词、解决方案词、产品词、选型词。

H2不能使用空泛标题，例如：案例板块、内容介绍、相关分析、产品优势、功能介绍、解决方案介绍。H2应让用户和搜索引擎能够直接理解该部分内容。

**重要：H2标题之间禁止使用横线（---）分割。** H2与H2之间、H2与H3之间直接通过空行衔接，不插入横线。横线仅用于输出信息部分（长尾关键词拓展、SEO信息）之间的分隔。

#### H3标题要求

H3不能只写“案例一”“场景一”“功能一”等缺少信息量的标题。H3需要包含行业关键词、工艺场景、产品参数、价值结果或选型判断点。

#### 正文内容要求

- 引言段落 1-2 段（在第一个 H2 之前，不用 H2 包裹），第一段直接回答主题问题，第二段说明选型或应用风险。
- H2/H3 层级清晰，每个 H2 解决一个真实搜索问题，每个 H3 拆解一个细分长尾词或具体判断点。
- 需要表格时使用 Markdown 表格。
- 案例标题必须包含行业、产品或场景关键词，不能写“案例一”。
- 文章整体建议 1800-2500 字；技术深度文章可扩展到 3000 字。

### 4. FAQ

根据主题复杂度输出 4-8 个 FAQ，最多不超过 8 个。

FAQ 问题类型应根据主题选择覆盖：

- 定义型：是什么、什么意思、包括哪些部分
- 选型型：怎么选、看哪些参数、适合哪些企业
- 参数型：负载怎么选、臂展多长合适、能码多高、精度怎么看
- 场景型：适合哪些行业、能不能做某个工艺、是否适合小批量多品种
- 对比型：和工业机器人、传统专机、人工方案有什么区别
- 落地型：如何部署、是否需要改产线、如何和 PLC/MES/视觉/夹具对接
- 成本型：多少钱、回本怎么评估、影响价格的因素有哪些
- 品牌产品型：艾利特哪个系列适合、ES/CS/CSA/CSH/CP/CW/Centaur-G1 怎么匹配
- 风险误区型：哪些场景不适合、选型常见误区、参数不能怎么理解

FAQ 要求：

- 问题贴近真实搜索习惯，本身就是长尾关键词载体。
- 问题中自然包含核心关键词、长尾关键词、场景词、产品词、参数词或对比词。
- 回答建议 80-150 字，直接、专业、可独立理解。
- 不要为了凑数量写重复问题。
- 不要每个 FAQ 都强行植入艾利特。
- FAQ 要补充正文没有完全展开的后续疑问。
- 不要写没有搜索价值的泛问题，如“有什么优势”“为什么重要”，除非已被具体化为真实长尾词。

### 5. 总结

总结 1-2 段。

要求：

- 回到标题问题。
- 给出清晰判断。
- 自然说明艾利特相关产品适合在哪些条件下评估。
- 不写“立即咨询、马上购买、行业首选”等硬广。

## 十一、术语表述规范

首次出现以下术语时，需要给出简洁解释：

- 协作机器人：强调安全、易用、柔性部署，可与人员在共享空间协同作业的机器人类型。
- 轮式人形机器人：采用轮式移动底盘和人形上肢结构，兼具移动、感知、双臂操作和任务决策能力的机器人形态。
- 具身智能：让机器人在真实物理环境中通过感知、理解、规划和动作执行完成任务的智能方式，重点不是“会聊天”，而是能与物理世界交互。
- 具身智能大模型：面向机器人感知、空间理解、操作规划和动作决策的大模型能力体系，写作时要说明其与具体传感器、控制系统和应用任务的关系。
- 多模态感知：融合视觉、深度、力觉、位置、语义等多种信息源理解环境和任务状态。
- 手眼力协同：机器人将视觉感知、机械臂运动和力觉反馈结合，用于抓取、插接、装配、搬运等需要精细交互的任务。
- SLAM 导航：机器人在未知或半已知环境中同步完成定位与地图构建，常用于移动底盘自主导航。
- 麦克纳姆轮：可支持全向移动的轮式结构，适合狭窄空间、跨工位移动和灵活路径调整。
- VLA 架构：Vision-Language-Action，视觉、语言和动作联合建模架构；写作时需解释为“把看见的环境、理解的任务和执行动作连接起来”，不要写成无条件自主完成所有任务。
- 负载：机器人末端可承载的工件、夹具和附加设备重量总和。
- 工作半径/臂展：机器人末端可覆盖的最大作业范围，影响托盘、工位和设备布局。
- 重复定位精度：机器人多次到达同一位置的一致性，影响装配、检测、焊接等工艺稳定性。
- 末端执行器：安装在机器人末端的夹爪、吸盘、焊枪、打磨工具、视觉设备等。
- 拖拽示教：操作人员通过手动拖动机器人记录轨迹或点位，降低编程门槛。
- 图形化编程：通过可视化界面配置机器人动作和逻辑，适合非专业编程人员。
- 六维力/力矩传感器：可感知三个方向力和三个方向力矩的传感器，适合力控装配、打磨、检测等场景。
- IP 防护等级：衡量设备防尘、防水能力的等级。
- MTBF：平均无故障工作时间，用于评估设备可靠性。
- 工艺包：针对码垛、焊接等特定工艺封装的参数、流程和操作工具。
- 离线仿真：在虚拟环境中验证机器人动作、路径和程序，减少现场调试时间。
- SDK：软件开发工具包，用于二次开发和系统集成。
- ROS2：常用于机器人开发的软件框架，适合科研和复杂系统集成。
- PROFINET、Ethernet/IP、MODBUS-TCP：常见工业通信协议，用于机器人与 PLC、产线设备、上位系统连接。

没有真实项目依据时，不要硬写具体百分比。可以使用以下表达：

### 3. 克制的数据表达方式

允许使用克制、条件化的数据表达：

- 在部分重复性强、节拍稳定的工序中，机器人方案通常更容易保持连续作业一致性。
- 对小批量多品种场景，图形化编程和工艺包的价值主要体现在换产配置、任务复用和人员培训环节。
- 对码垛场景，可从每分钟处理箱数、托盘尺寸、最高堆高、产品换型时间和单班稳定运行情况评估方案效果。
- 对焊接场景，可从焊缝一致性、返修率、工装定位、编程时间和人员上手周期评估方案效果。

### 4. 禁止的数据表达方式

禁止写以下类型的数据：

- 没有依据的“效率提升 80%”“成本降低 60%”“3 个月回本”“节省 10 个工人”。
- 没有依据地写“一台焊接机器人通常替代 1-2 名焊工”或其他自行估算的人工替代数量。
- 把单个案例数据写成普遍承诺。
- 只写“提升效率 XX%”，但不说明工序、原始状态、对比对象、统计口径和适用条件。
- 使用“最高、最快、最低、最省”等绝对化数据表达，除非资料明确给出且语境必须使用。
- 为了吸引点击编造 ROI、回本周期、产能提升、良率提升、故障率下降等数字。

推荐表达：

不推荐表达：

## 十六、样例主题验证规则

使用本提示词生成文章时，可用以下 4 个主题快速检查产品匹配和 SEO/GEO 结构是否正确：

### 1. 协作机器人怎么选

期望输出：

- 核心关键词应围绕“协作机器人怎么选”或“协作机器人选型”。
- 正文应覆盖负载、臂展、重复定位精度、安全、编程方式、末端工具、集成接口、维护和应用场景。
- 产品匹配应包含 LS、ES、CS、CSA 系列的差异，不应只泛泛推荐“艾利特协作机器人”。
- 内容应包含选型对比表格和可引用的选型结论；FAQ 不超过 8 个，但要覆盖关键长尾问法。

### 2. 码垛机器人适合哪些行业

期望输出：

- 核心关键词应围绕“码垛机器人”或“码垛机器人适合哪些行业”。
- 正文应覆盖食品饮料、日化、化工、包装印刷、电商仓储、快递物流、新能源等场景。
- 产品匹配应优先关联 CSH 地平线系列和 CP 系列码垛一体机。
- 内容应解释码垛工艺包、自动排样、3D 实时预览、多配方垛型切换、快速部署、负载、工作半径、托盘尺寸、堆垛高度和换产效率的选型意义。
- FAQ 应承接“码垛机器人能码多高、多少钱、适合什么箱规、是否需要改产线、CP 和 CSH 怎么选”等潜在长尾需求。

### 3. 焊接工作站和传统焊接设备有什么区别

期望输出：

- 核心关键词应围绕“焊接工作站”或“机器人焊接工作站”。
- 正文应对比传统人工焊接、传统焊接专机和协作机器人焊接工作站。
- 产品匹配应优先关联 CW 系列焊接工作站、全功能焊接工艺包、Co-Pilot Tool 智能编程工具、智能免示教焊接系统和 CS66/CS612 等可选机器人。
- 内容应覆盖小批量多品种、焊缝一致性、软件包降低编程门槛、免示教减少逐点调试、工装定位、图形化编程、培训成本、CWT/CWC/CWP/CWG/CWF 场景选型、柔性扩展和安全部署。
- FAQ 不超过 8 个，但要覆盖“机器人焊接适合小批量吗、焊接工作站多少钱、是否需要专业焊工、和传统专机区别”等关键追问。

### 4. 人形机器人适合哪些场景

期望输出：

- 核心关键词应根据主题和搜索意图判断，可以围绕“人形机器人”“轮式人形机器人”“AI 人形机器人”“智能人形机器人”“人形机器人应用场景”等方向，但不要被这些示例词机械限定。

- 产品匹配应优先关联艾利特人形机器人 / Centaur-G1 轮式人形机器人；元启 Primo 可作为艾利特“一脑多形”和具身智能技术体系背景出现，不作为独立产品推荐。

- FAQ 不超过 8 个，但要覆盖“人形机器人和复合机器人区别、轮式人形机器人适合哪些场景、能不能用于工业巡检、参数怎么看、是否适合医疗健康或家庭服务”等关键追问。
